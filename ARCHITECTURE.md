# ARCHITECTURE.md - System Map

## 1. Overview
The AI Context Protocol Engine (`aigit`) serves as the central intelligence coordination layer for the workspace. It enforces memory persistence and agent skill constraints using file-based schemas, with long-term memory managed via a contextual pgvector database and semantic commit hooks.

## 2. Agent Directory (`.agent/`)
This directory holds the identities, constraints, and operational protocols of all specialized agents. The system defines 8 available master agents, with the core set currently materialized:

- **`.agent/orchestrator.md`**: Multi-agent coordination and synthesis.
- **`.agent/project-planner.md`**: Discovery, Architecture, and Task Planning. Enforces the 4-Phase methodology (Analysis, Planning, Solutioning, Implementation). No code is authored by this agent.
- **`.agent/backend-specialist.md`**: Backend Architect (API + Database + Server/Docker Deploy).
- **`.agent/frontend-specialist.md`**: Frontend & Growth (UI/UX + SEO + Edge/Static Deploy).
- **`.agent/security-auditor.md`**: Master Cybersecurity (Audit + Pentest + Infra Hardening).
*(Additional agents like `mobile-developer`, `debugger`, and `game-developer` are routed dynamically based on domain).*

## 3. Modular Skills (`.agent/skills/`)
Skills are independently loadable cognitive layers that extend agent capabilities:

- **`clean-code`**: Universal coding standards (GLOBAL mandate).
- **`brainstorming`**: Socratic Gates and problem exploration.
- **`frontend-design`**: Web UI patterns ensuring distinctive, production-grade interfaces.
*(Other specialized skills mapped in the system include `database-design`, `app-builder`, `mobile-design`, `plan-writing`, `threejs-mastery`, and `behavioral-modes`).*

## 4. Workflows & Scripts (`.agent/scripts/` & `scripts/`)
Defines standardized sequences of commands and auditing tools for the Final Checklist Protocol and Slash Commands:

- **Root Scripts**: `scripts/clean_drift.py` for context drift resolution.
- **Audit Checklist**: `.agent/scripts/checklist.py` for manual and pre-deploy auditing.
- **Compiler**: `.agent/scripts/compiler.js` for agent pipeline processing.

## 5. Persistence Layer (`context-server/`)
The project utilizes a Node.js Context MCP Server for semantic memory and state preservation.
- **`context-server/prisma/schema.prisma`**: The database schema representing memories, agents, sessions, and tasks mapping to the local PGLite/PostgreSQL database.
- **Smart Hybrid Git Hooks**: A non-bypassable semantic commit hook (enforced via `cli/index.ts` and `aigit commit memory`) that prevents AI agents from skipping memory generation while gracefully prompting human developers.

*Note: For the detailed autogenerated semantic causal graph and task history of the Context Server itself, refer to `context-server/ARCHITECTURE.md` (generated via `aigit docs`).*
