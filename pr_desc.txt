💡 What: Added `role="list"` and `role="listitem"` to UI lists in Conflicts and Search pages, and added an `.sr-only` utility class to provide context for visually ambiguous content.
🎯 Why: Screen reader users previously heard a series of `<div>` elements without list grouping or semantic context for memory and search result contents. This provides structural meaning and screen-reader specific labels.
📸 Before/After: Visual UI remains unchanged. The `.sr-only` class hides text visually while exposing it to assistive technologies.
♿ Accessibility: Improves WCAG compliance for lists (providing explicit roles when non-semantic elements are used) and provides necessary context for screen reader users when parsing visually implied blocks of text.
