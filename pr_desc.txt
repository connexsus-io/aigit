💡 What: Added accessible loading states (dynamic aria-label, aria-busy, disabled) and a visual spinner to the 'Retry Search' button on the Search page.
🎯 Why: Error retry buttons trigger background fetches just like primary actions, but the previous implementation lacked loading states. This meant screen readers didn't know the fetch was occurring, and users could spam-click the button.
📸 Before/After: N/A - Visual change is only during the loading state, which now matches the primary search button.
♿ Accessibility: The button now announces its loading state properly to screen readers and prevents multiple rapid submissions.
