💡 What: Added screen-reader-only labels to visually generic metadata (like file paths and agent names) in the Conflicts and Search pages. Also added an `aria-live` summary to announce the number of conflicts found.
🎯 Why: Without these labels, screen readers would simply read out raw file paths and names without explaining their significance in the layout. Providing an `aria-live` summary for the Conflicts page prevents the list itself from being completely scanned unnecessarily while keeping users informed of the scope of their work.
📸 Before/After: Visuals remain unchanged, but screen readers now accurately understand context.
♿ Accessibility: Improves WCAG compliance regarding structural meaning and status announcements.
